# Development Process

This folder contains process documentation for auto-dev framework.

## Process Flow

```
Version Scoping → Requirements → Implementation Plan → Starter Prompt → Implementation → Retrospective → Version Close
```

## Documents

### Generic Process (Synced)

These documents are synced from the auto-dev template source:

| Document | Purpose |
|----------|---------|
| [01-VERSION-SCOPING.md](./generic/01-VERSION-SCOPING.md) | Define version scope |
| [02-REQUIREMENTS.md](./generic/02-REQUIREMENTS.md) | Document requirements |
| [03-IMPLEMENTATION-PLAN.md](./generic/03-IMPLEMENTATION-PLAN.md) | Plan implementation |
| [04-STARTER-PROMPT.md](./generic/04-STARTER-PROMPT.md) | Generate starter prompt |
| [05-IMPLEMENTATION.md](./generic/05-IMPLEMENTATION.md) | Implementation guidance |
| [06-RETROSPECTIVE.md](./generic/06-RETROSPECTIVE.md) | Post-implementation review |
| [07-VERSION-CLOSE.md](./generic/07-VERSION-CLOSE.md) | Close out version |

### Specialized Process (Local)

Create `specialised/` folder for project-specific process documents.
These are never overwritten by template sync.

## Usage

1. Start with version scoping
2. Follow documents in order
3. Customize specialized documents as needed
4. Capture learnings in LEARNINGS.md
